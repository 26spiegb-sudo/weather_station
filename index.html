Weather Station Project

Engineering Design Portfolio

Description

This project is an Arduino-based weather station that measures and displays temperature, humidity, barometric pressure, and maximum/minimum temperature in real time. The device uses a BME280 sensor and outputs readings to a TFT display shield mounted on an Arduino Mega. All components are in a custom-designed, 3D-printed enclosure created using Fusion 360 and printed on a Bambu printer. A successful project displays accurate, live environmental data in a clear, readable format.

Inspiration: Arduino Easy Weather Station With BME280 Sensor by Nick Koumaris

Differences from the source: The original project uses a 16x2 LCD and shows readings every 2 seconds. This project uses a TFT color display, cycles through readings every 3 seconds, displays in Fahrenheit, tracks max/min temperatures, and is enclosed in a custom 3D-printed case.

Design considerations: All electronics were measured precisely using calipers for the enclosure. The sliding mechanism allows easy access for maintenance. Iterative printing ensured proper fit, visibility, and durability.

Final Summary

This project resulted in a fully functional Arduino-based weather station capable of measuring and displaying real-time temperature, humidity, and barometric pressure through a TFT color display, housed in a custom 3D-printed enclosure. The process of building it required navigating a series of hardware and software challenges — from sensor compatibility issues to iterative enclosure redesigns — that pushed the project well beyond the scope of the original tutorial. Each obstacle required independent research, creative problem-solving, and a willingness to restart portions of the work from scratch.

One of the most significant technical challenges was display compatibility. The first display was found to be incompatible with the Arduino Mega after several days of troubleshooting, requiring a replacement to be ordered. When the new display arrived, it introduced a new problem by blocking the sensor's pin connections, which required code-level rewrites to resolve. These setbacks were instructive — they demonstrated that hardware integration is rarely straightforward and that troubleshooting is an essential, time-consuming part of the engineering process rather than an exception to it.

The enclosure design went through six printed iterations over the course of two weeks. Each version revealed new issues — walls too thin, display misaligned, cable hole off by a fraction of an inch — that required design updates in Fusion 360 before reprinting. The sliding rail mechanism that was developed to close the case was a design solution that emerged directly from the constraints of the build, rather than from the original plan.

Managing a project of this scale over several months also required consistent organization. Using GitHub for version control, maintaining detailed documentation, and tracking decisions as they were made all proved to be as important as the technical work itself. The project also highlighted the value of AI tools — used responsibly to assist with debugging and code customization — as a practical resource in the engineering workflow.

Looking ahead, several improvements would strengthen the project. Adding ventilation holes around the BME280 sensor would allow better airflow and improve measurement accuracy, as heat from the enclosed electronics was observed to skew temperature readings over time. Tighter tolerances in the enclosure would eliminate the remaining display movement. A longer-term enhancement could include Wi-Fi connectivity via an ESP8266 or similar module, enabling the station to log data over time or push readings to a web dashboard — expanding the project from a standalone display into a networked environmental monitoring tool.

Mistakes / Solutions

During assembly and programming, the first display was incompatible with the Arduino Mega, and the sensor was blocked by the new display's pin layout. These required code rewrites and a display replacement.

Design iterations revealed issues with wall thickness, cable holes, and sliding mechanism tolerances, which were corrected over multiple prints.

Reflection

This project taught me the importance of troubleshooting, research, and iterative design. Hardware integration is rarely straightforward and requires patience and problem-solving.

I learned how to work with Arduino, sensors, 3D printing, and version control, combining software and hardware effectively.

The daily journal helped me track progress, and consistent documentation proved invaluable for resolving issues efficiently.

Iterating on the enclosure design taught me precision, measurement, and adaptation skills critical to large projects.

For future projects, I plan to incorporate wireless data logging and improved ventilation, continuing to refine my engineering workflow and build confidence in tackling complex systems.
